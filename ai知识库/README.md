# AI Knowledge Base（骨架）

配套同级 `../运营自动化项目` 的共享 AI 知识库**骨架**。目录结构与机制已就位，
但**不含任何业务知识正文**——正文由运营项目的 `ai_knowledge_base_update`
workflow 自动生成填充。

## 目录结构

| 目录 | 用途 |
|---|---|
| `00-总览/` | 版本、系统能力地图、工作流总览、命令映射表、当前项目状态（Hermes 读取入口）|
| `01-工作流/` | 每个 workflow 一个 `.md`，frontmatter 是路由信息的唯一真相源 |
| `02-平台能力/` | 平台层（Ops-Cli）能力清单 |
| `03-SOP/` | 标准作业流程 |
| `04-项目文档/` | 架构 / 边界 / 集成说明 |
| `05-运行报告/` | 运行记录摘要 |
| `06-需求池/` | 待办需求 |
| `07-提示词/` | 可复用提示词模板 |
| `08-决策记录/` | ADR 架构决策记录 |
| `99-归档/` | 废弃 / 下线内容归档 |

## 怎么用

1. 把同级运营项目跑起来（见 `../运营自动化项目`）。
2. 运行 `ai_knowledge_base_update` workflow 生成 / 刷新本库正文（见 `SYNC_RULES.md`）。
3. 运行 `scripts/validate_ai_knowledge_base.py` 校验。

机制与红线详见 `AGENTS.md` 与 `SYNC_RULES.md`。
